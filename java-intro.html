What is Java?

Java is an Object-Oriented, Compiled programming language that is syntactically very similar to C.

Why Java?

  • Consistently one of the top 5 languages for the last 20 years
  • Considered a “step up” in functionality from plain C
  • Provides a “write once, run anywhere” approach which is enabled by the use of the Java virtual machine (JVM)
  • Automatic memory management 1
  • Has the slight edge in offering higher-level multi-threading constructs OOTB 2

Anatomy of a Basic Program

A basic Java program is slightly different from a Python or C program. 

public class HelloWorld
{
    public static void main(String [] args)
    {
        System.out.println("Hello, world!");
    }
}

Classes

A class is a way of grouping related functions or methods.

A class is defined as follows: 

public class HelloWorld
{
    // Here's a bunch of methods and fields!
    // ...
}

In general, class declarations can include these components in order, which we will talk about further in What is Object-Oriented Programming?

  1. Modifiers: A class can be public or have default access Access Modifiers.
  2. Class name: The class name should begin with the initial letter capitalized by convention.
  3. Superclass: The name of the class’s parent (superclass), if any, denoted by extends. A class can only have one parent.
  4. Interfaces: A comma-separated list of interfaces implemented by the class, if any, denoted by implements. A class can implement more than one interface.

NOTE: If a class is named HelloWorld, then the file must also be named HelloWorld.java, case matched.

The main() Method

The main method signature is how the Java file runs, much like in C. 

    public static void main (String [] args)
    {
        // ...
    }

Unlike C:

  • Every method, including the main method, must be a part of a class.
  • The return type is void and not int
  • Java’s main() method can input an array of strings that correspond to command line arguments by default.

System Output

There are three methods generally used to produce system output. 

    System.out.println("1");
    System.out.print("2\n");
    System.out.printf("3\n");

System.out.printf() is only used if the advanced formatting of printf is needed. otherwise, print() or println() are used.

String Concatenation

Concatenating strings for printing in Java is as easy as in Python. 

    System.out.println("Check " + "out " + "these " + "sentences!");

Compiling Programs

Compiling programs is fairly simple. Given as the polyoculus code has all of the files compiled using a program called Make, you won’t have to do this step unless you create a new file that is not recognized by the makefiles. However, if you need to create and compile a new file, it’s as easy as 

# This command compiles the .java file into a bytecode file with the extension .class.
javac HelloWorld.java
# This command then compiles and runs that .class file in the Java virtual machine.
java HelloWorld

NOTE: When compiling the source code with javac, you need to include the file extension. When running the compiled code, you do not.

Compiling Multiple Source files into One Program

    {

If we compile and run SetUp.java

    So it opens that file and compiles it along with SetUp.java. That’s why Java requires us to write public classes in files with their corresponding names.

Primitive Data Types in Java

In Java, each primitive data type also has a corresponding Class. Each of these classes supplements the primitive type with various methods and predefined values. They are all very well documented on the javadocs, so we will explore just one. We will also discuss strings, as they are a different in Java than in languages like C.

Integers

int

In most3 programming languages, an int is defined as 32 bits by default. But, that can vary from system to system. There is no guarantee that an int will be 32 bits on all systems. This is historically a problem with languages like C.

This is an issue if you need to compute an integer that may be greater or less than \(\approx\) 33,000 and integers are 16-bit.

Java solves this by compiling code first to byte-code in a .class file, which then gets executed by the JVM to ensure that the program runs the same on any machine.

The Integer Class

In contrast with the primitive int type, Integer is a built-in class that has a number of useful methods such as the maximum and minimum values for an integer. 

    System.out.println(Integer.MAX_VALUE);
    System.out.println(Integer.MIN_VALUE);

There are some other syntactic differences as well, such as comparing values: 

    int primitive = 10;
    Integer fancy = 10;
    System.out.println(primitive == 10);
    System.out.println(fancy.equals(10));

Strings

In C, a string is an array of characters. In Java, strings are immutable objects. Immutable objects mean that once they are created, they cannot be changed. This means that every time you create a string, it does not modify the previous string, it creates a new one. 

    // This is how you would make an array in a language like C.
    char[] helloChar = {'h', 'e', 'l', 'l', 'o'};
    String helloString = new String(helloChar);
    // Check the type of each "string"
    System.out.println(helloChar.getClass().getName());
    System.out.println(helloString.getClass().getName());

This immutable property is important as it affects the runtime of the program, as every time you create a string, it is an O(k) operation, where k is the length of the string. If you are doing this operation many thousands of times, it can slow down your program significantly.

Data Structures

Arrays

Arrays are very much like lists in Python. In Java, all arrays are dynamically allocated, and are stored contiguously. Arrays can be of any object type, and can be n-dimensional.

Java auto-initializes all array values and class fields to 0.

An array in Java is initialized in a specific way: 

    char [] array = new char[15];
    System.out.println(array.length);

Command-Line Arguments

A cool part of the main() function in Java is that, by default, you can pass command-line arguments and read them quite simply. 

public class CommandLineArgs
{
    public static void main(String [] args)
    {
        for (int i = 0; i < args.length; i++)
            System.out.println(args[i]);
    }
}

javac CommandLineArgs.java
java CommandLineArgs check out these arguments!

ArrayLists

An ArrayList is a resizable object array that is contiguous in memory, much like an array. In fact, it is almost identical to an array, save for the fact that it implements the List interface. 

    // Note the specification of the type of arraylist here, in brackets.
    ArrayList<String> trucks = new ArrayList<String>();
    trucks.add("Ford Super Duty");
    trucks.add("Toyota Tundra");
    trucks.add("RAM 1500");
    trucks.add("Toyota Tacoma");

    for (String truck : trucks)
        System.out.println(truck);

NOTE: Make sure to import java.util.ArrayList.

Linked Lists

In Java, unlike in C, one can call a LinkedList without having to make one from scratch. This is really handy, as it has already been optimized for usage and doesn’t have to be rewritten. The LinkedList and the ArrayList classes have the same methods as they both implement the List interface. This the add, change, and remove methods are called in the same way. 

    LinkedList<String> listy = new LinkedList<>();
    listy.add("Hello");
    listy.add("World!");

    for (String list : listy)
        System.out.println(list);

Errors and Exceptions

In Java, there are two types of issues that may arise with a program, errors and exceptions. Both errors and exceptions are subclasses of the java.lang.Throwable class.

Errors are serious problems that are outside of control of the program, like a memory error, or a stack overflow error. Often, Errors cannot be found until the program is compiled and run.

Exceptions are related to issues that can be recovered from within the program. This includes things like accessing an array out of bounds, sending an illegal argument to a method, There are two types of “errors” that occur with Java.

Flow Control

    try
    {
        // ...
    }
    catch(Exception e)
    {
        System.out.println("Oops! There was an exception!");
    }
    finally
        // ...

Checked vs Unchecked Exceptions

Checked exceptions are the exceptions that are caught at compile time. Within checked exceptions, there are fully checked and partially checked exceptions.

Unchecked exceptions are not checked and found at compile time. Unchecked exceptions are things like NullPointerException, where the program is trying to access a null pointer in memory.

User-Defined Exceptions

Sometimes there are instances in which something will break in a way that is very specific to the program. This might mean that there is nothing built-in to Java to check when an exception may occur, and the programmer may need to create their own.

Good Coding Practices

Naming Conventions for Variables

In Java programming, there are a few important naming conventions that should be followed for legibility.

You will find in multiple locations in the legacy code where this is not followed; although it would be most appropriate to fix every one of these instances, it would become a large undertaking to do this for every package used.

For the code directly related to polyoculus, these naming conventions should be followed, and for things like the INDI package, it isn’t necessary to fix, since this is beyond the scope of our project.

These are all naming conventions that are used throughout the “Java-speaking” world, and you can find the full naming convention document here.

  • All Java variables are named in camelCase, where each portion of the word is capitalized without spaces. In Python, names are usually in snakecase, where all variables are in lower case and separated by underscores.
  • Classes should be nouns with their first letter capitalized. They should be simple and adequately describe the class.
  • Methods should be verbs, which the first letter of each subsequent word is capitalized.
  • Constants should be in all caps, with underscores delineating spaces.

Below is a simple example using all of these naming conventions: 

public class NameExample
{
    static final int MAX_RAND = 100;

    private static int generateNumber()
    {
        return (int)(Math.random() * MAX_RAND + 1);
    }

    public static void main(String [] args)
    {
        System.out.println(generateNumber());
    }
}

Avoiding “Magic Numbers”

Magic numbers present a multitude of problems for both program writers and readers.

  • Hard-coding a number obfuscates the purpose of that number.
  • Updating magic numbers becomes difficult when they are used in multiple places within a codebase. For our purposes, “magic variables” become an almost impossible task to track down.

A bad example of having a magic number, in this instance within a for loop. 

    int array [] = new int[10];
     ...
    for (int i = 0; i < 10; i++)
        System.out.println(array[i]);

A good example of removing magic numbers from the code by calling the array’s builtin .length function. 

    int [] array = new int[10];
    for (int i = 0; i < array.length; i++)
        System.out.println(array[i]);

Style Guide

According to the Java specification from Oracle, these are some spacing rules:

  • A blank space should appear after commas in argument lists.
  • All binary operators except . should be separated from their operands by spaces.
  • Blank spaces should never separate unary operators such as unary minus, increment (“++”), and decrement (“–”) from their operands.

Curly Braces

There are a few options for writing curly braces for methods and classes. The most common and preferred method is the One True Brace Style or OBTS. This is also sometimes referred to as “Egyptian Braces” style. 

public void checkNegative(x) {
    if (x < 0) {
        System.out.println("Negative");
    } else {
        nonNegative(x);
    }
}

Another style is the K&R style, which is the standard style used in C. It places the curly braces on separate lines, which helps delineate code blocks. In K&R, single line blocks do not have curly braces. 

public void checkNegative(x)
{
    if (x < 0)
    {
        absx = x * -1;
        System.out.println("Negative");
    }
    else
        nonNegative(x);
}

For methods that only have a single line, it is appropriate to condense the method as follows: 

    private Object obj;

    public Object get(){ return this.obj; }
    public void set(Object obj){ this.obj = obj; }

What is Object-Oriented Programming?

The four main principles of Object-Oriented Programming (OOP) are:

  • Abstraction
  • Inheritance
  • Encapsulation
  • Polymorphism

We will take a look at all of these in depth.

What is an Object?

An object is the atomic unit of the OOP paradigm that represents the real entities interacting within a program. The objects interact with each other by invoking methods. An object consists of three main components:

  • State, representated by attributes (i.e. fields)
  • Identity, which is the uniqueness property that allows an object to interact with other objects
  • Behavior, the actions performed by the methods within an object.

The most common example used for explaining objects and OOP in education is a car. A car requires the participation of many independent objects working in tandem with each other to function.

Abstraction

In Java, abstraction is accomplished through the implementation of interfaces and abstract classes.

Abstract Classes

An abstract class is a restricted class that is used as a template only, and cannot create objects by itself. It may or may not include abstract methods within the class. 

abstract class Creature
{
    public abstract void creatureSound();

    public void sleep()
    {
        System.out.println("Snrk. Mimimimi... zzzz...");
    }
}

If you were to try and create a Creature object, Java would produce an error. 

Creature cat = new Creature(); // This will generate an error.

Interfaces

An interface is a completely “abstract” class also used to group related methods, but with empty bodies, unlike abstract classes.

An interface, unlike an abstract class, is like a template where the method signatures are provided and the subclass must fill in the methods.

Here is an example, where there are multiple interfaces, and a class that extends them both. 

interface Creature
{
    public void sleep();
}

interface Pet
{
    public void setVaccineStatus(boolean update);
}

class Cat implements Creature, Pet
{
    private boolean vaccine = false;

    public void setVaccineStatus(boolean update)
    {
        this.vaccine = update;
        if (this.vaccine == true)
            System.out.println("the vaccines are up to date!");
    }

    public void sleep()
    {
        System.out.println("Zzzz... zzzz...");
    }
}

class Main
{
    public static void main(String [] args)
    {
        Cat saavik = new Cat();
        saavik.sleep();
        saavik.setVaccineStatus(true);
    }
}

Zzzz... zzzz...
the vaccines are up to date!

Inheritance

Inheritance in Java and other OOP languages is the idea that an object can inherit, or acquire, all of the properties and behaviors of its parent, sort of like in real life. A class can only extend one other class, as Java does not support multiple inheritance.

When >= 2 methods in the same class have the same method name but different parameters, this is called overloading. When two methods have the same method name and parameters, this is called method overriding.

Classes and Subclasses

Let’s take a look at a class for a Person object, which has multiple fields and a constructor method. 

public class Person
{
    public String name;
    public int age;

    public Person(String name, int age)
    {
        this.name = name;
        this.age = age;
    }

    public String toString()
    {
        return name + " is " + age + " years old";
    }
}

Now, let’s create a child class, that extends the parent class. 

public class Student extends Person
{
    public int studentID;

    public Student(String name, int age, int studentID)
    {
        super(name, age);
        this.studentID = studentID;
    }

    public String toString()
    {
        return super.toString() + "; student id: " + studentID;
    }

    public static void main(String [] args)
    {
        Person jackie = new Person("Jackie", 22);
        Student alex = new Student("Alex", 22, 123456);

        System.out.println(jackie.toString());
        System.out.println(alex.toString());
    }
}

There’s a lot going on here!

The class we’re extending is called the “superclass” or “parent class.” The class that’s doing the inheriting is called the “subclass,” “child class.” We say the subclass extends or “inherits from” the superclass. We can also say “the subclass is a superclass.” In this example, the Student is a Person.

Inheritance allows us to form hierarchies of inheritance, or “taxonomies” like in biology.

The child class inherits all fields and methods from the parent class except for private fields and methods. Those are the parent’s secrets, and they are not passed on to the child.

Encapsulation

Because objects make decisions and enact logic based on their state, we want to make sure that each object manages its own state and protect against state mutations from other outside classes.

With encapsulationm, the variables or data of a class is hidden from any other class and can be accessed only by what’s called colloquially as setter and getter functions.

Some advantages of encapsulation:

  • Data obfuscation
  • Reusability
  • Ease of testing
public class EncapsulatedStudent
{
    private String name;
    private String studentID;
    private int age;

    public int getAge()
    {
        return age;
    }

    public String getName()
    {
        return name;
    }

    public void setAge(int age)
    {
        this.age = age;
    }

    public void setName(String name)
    {
        this.name = name;
    }

    public static void main(String [] args)
    {
        EncapsulatedStudent student = new EncapsulatedStudent();
        student.setName("Alex");
        student.setAge(22);

        System.out.println(student.getName() + " aged " + student.getAge());
    }
}

Alex aged 22

NOTE: Stylistically, a setter method and a getter method should both start with set and get, respectively.

Polymorphism

In Java, polymorphism provides different implementations of a method depending on the type of object passed to that method.

Types of Polymorphism

You can perform Polymorphism in Java via two different methods:

  • Method Overloading
  • Method Overriding

Method overloading is the process that can create multiple methods of the same name in the same class, and all the methods work in different ways.

This is best shown as an example with method constructors: 

public class Shape
{
    public int length;
    public int width;
    public int radius;
    public final double pi = 3.1415926535;

    public Shape(int length, int width)
    {
        this.length = length;
        this.width = width;
    }

    public Shape(int radius)
    {
        this.radius = radius;
    }

    public static void main(String [] args)
    {
        Shape rectangle = new Shape(10, 6);
        Shape circle = new Shape(4);

        System.out.println("The area of the rectangle is " + rectangle.length * rectangle.width);
        System.out.println("The area of the circle is " + circle.radius * circle.pi);
    }
}

Here there are two different constructors, one for a circle and one for a rectangle.

Polymorphism with Inheritance

A superclass named Shapes has a method called area(). Subclasses of Shapes can be Triangle, Circle, Rectangle. Each subclass has its way of calculating area. 

class Shapes
{
    public void area()
    {
        System.out.println("The formula for area");
    }
}

class Triangle extends Shapes
{
    public void area()
    {
        System.out.println("Triangle is ½ * base * height ");
    }
}

class Circle extends Shapes
{
    public void area()
    {
        System.out.println("Circle is 3.14 * radius * radius ");
    }
}

    Shapes shape = new Shapes();
    Circle circle = new Circle();
    Triangle triangle = new Triangle();

    shape.area();
    circle.area();
    triangle.area();

NOTE: when there are multiple classes in the same file, you should not make the classes public.

Access Modifiers

  • public: Accessible in all classes in your application.
  • protected: Accessible within the package in which it is defined and in its subclass(es) (including subclasses declared outside the package).
  • private: Accessible only within the class in which it is defined.
  • default (declared/defined without using any modifier): Accessible within the same class and package within which its class is defined.

Generic Types

We can create a basic class that we want to hold any type. Since we know that everything in Java is a subclass of the Object class, we can assume that the best, most generic course of action is to have the box store an object. 

public class Box {
    private Object object;

    public void setContents(Object object) { this.object = object; }
    public Object getContents() { return object; }
}

This code is very problematic! There is no way to verify at compile time how the class is going to be used. One section of the code may set an Integer into the box contents and expect to get Integers out of it, while another part of the code may pass in a String, which will throw a runtime error. It also gives no information to the programmer as to what the box may be used for.

To solve this, we can create what is called a generic class. To instantiate a generic class, we type 4

With this change, the Box class becomes: 

/**
 * Generic version of the Box class.
 * @param <T> the type of the value being boxed
 */
public class Box<T> {
    // T stands for "Type"
    private T t;

    public void setContents(T t) { this.t = t; }
    public T getContents() { return t; }
}

Instantiating Generics

Whenever instantiating a generic class, the type has to be specified within angle brackets. 

    // In modern Java, you only have to specify the type once.
    // Leave the second pair of angle brackets empty in what is called "the diamond".
    Box<String> stringBox = new Box<>;
    // In older versions of Java, this line would be instantiated as follows:
    Box<Integer> intBox = new Box<Integer>();

Multithreading

What is a Thread?

A thread is the atomic unit of programming instructions that can be handled independently by a scheduler5, which is part of the computer’s operating system.

Utilizing the Thread Class

All modern operating systems support concurrency through processes and threads. Normally, programs do not take advantage of concurrency, and run on a single thread. 

class MultiThreading extends Thread
{
    public void run()
    {
        // Let's wrap this in a try-catch block in case something weird happens.
        try {
            // Display the thread that is running
            System.out.println("Thread "
                + Thread.currentThread().getId()
                + " is running");
        }
        catch (Exception e) {
            System.out.println("Uh oh, something happened!");
        }
    }
}

class Main
{
    public static void main(String [] args)
    {
        int cores = Runtime.getRuntime().availableProcessors();
        for (int i = 0; i < cores; i++)
        {
            Thread thread = new Thread(new MultiThreading());
            thread.start();
        }
    }
}

As mentioned in Inheritance, Java does not support multiple inheritance. This can cause problems if a program is extending the Thread class. To get around this, you can instead implement the Runnable interface, which allows for thread creation and manipulation without extending a parent class.

The process of writing the function is the same, except now we can implement multiple classes and extend a parent class, if we need. 

class MultiThreading implements Runnable
{
    public void run()
    {
        try {
            System.out.println("Thread "
                + Thread.currentThread().getId()
                + " is running");
        }
        catch (Exception e) {
            System.out.println("Uh oh, something happened!");
        }
    }
}

class Main
{
    public static void main(String [] args)
    {
        int cores = Runtime.getRuntime().availableProcessors();
        for (int i = 0; i < cores; i++)
        {
            Thread thread = new Thread(new MultiThreading());
            thread.start();
        }
    }
}
